# CLAUDE.md — CFG Cost Lab 프로젝트 메타 문서

## 프로젝트 정체성

**CFG Cost Lab**은 카페게이트 구매물류팀 전용 원가율 진단·시뮬레이션 도구다.  
목표: 매출원가율 70%대 → **65%** 인하. 분석 축: 협력업체 매입가 vs CJ 역매입가.

## 사용자 프로파일

| 구분 | 인물 | 용도 |
|------|------|------|
| 1차 | 장인호 MD (구매물류팀) | 직접 사용, 분석 실행 |
| 2차 | 구매물류팀 동료 | 협업 검토 |
| 3차 | 본부장 | 보고 검토 |

- 인호님은 AI·코딩 초보자 → 설명은 중학생 수준, 명령어는 항상 코드 블록으로 명확히 제시
- 표·단계별 안내 선호

## 기술 스택

- **언어**: Python 3.11+
- **웹**: Streamlit
- **데이터**: pandas, openpyxl
- **시각화**: Plotly
- **저장소**: 엑셀 직접 읽기 (Phase 4에서 SQLite 검토)

## 핵심 계산 공식

```
매입 원가율   = (매입 단가 / 판매 단가) × 100
역매입 원가율 = (CJ 역매입 단가 / 판매 단가) × 100
비정상 플래그 = 원가율 > 80% (기본 임계값)
```

## Phase 로드맵

- **Phase 1 (현재)**: 품목별 원가율 전수조사 + 20·80 핵심품목 분석
- **Phase 2**: 점주 판가 가이드라인 시뮬레이터
- **Phase 3**: 유통구조 재설계 비교 (수수료 vs 직매입)
- **Phase 4**: 메뉴 원가 + 신메뉴 BEP 분석

## 보안 규칙 (절대 위반 금지)

1. `data/raw/` → `.gitignore` 필수 포함
2. 실제 거래처명·단가 → 코드/문서에 하드코딩 금지
3. 샘플 데이터 → 익명화 (A사·B사, 가짜 숫자)
4. 외부 클라우드 업로드 기능 추가 금지
5. 사내 PC 또는 사내 서버에서만 실행

## 코딩 규칙

- 함수·변수명: 영문 snake_case
- 주석: 한국어 허용 (팀 공유 코드)
- 마법 숫자 금지 → 상수로 분리 (예: `ABNORMAL_THRESHOLD = 80`)
- Phase 1 데이터 구조는 B/C/D 후속 프로젝트와 호환되게 설계

## 응답 규칙
